FR16

Import Products / Functional / Equivalence Partitioning

test-design/EP-FR16.md

| Coverage Type | Covered IDs |
|---|---|

## Detail
| ID | Test Objective | File CSV (Input) | Database State (Output) | UI Report (Output) |
|---|---|---|---|---|
| FR16_EP_V01 | Import file CSV hợp lệ hoàn toàn với các dòng dữ liệu đúng chuẩn | File `.csv` (EXT1), Header đúng (HD1), bọc nháy kép chuẩn (FMT1), tất cả các dòng dữ liệu hợp lệ (NAME1, PRICE1, CAT1, STATE1) | Thêm thành công toàn bộ sản phẩm vào DB (DB1) | Hiển thị thông báo thành công: 2/2 sản phẩm được thêm (REP1) |

## Preconditions
1. Người dùng đăng nhập hệ thống với tài khoản Admin (`role = 'admin'`).
2. Người dùng đang ở màn hình Import sản phẩm từ file CSV.
3. Có sẵn file CSV hợp lệ chứa các sản phẩm cần import.

## Test data
| Field | Value |
|---|---|
| File Name | products_valid.csv |
| File Content | name,price,description,imageUrl,category_id<br>iPhone 15 Pro Max,30000000,Điện thoại Apple,,1<br>"Bàn phím cơ, Keychron Q1",4000000,Bàn phím cơ cao cấp,,3 |

## Test steps
1. Chọn file `products_valid.csv` để tải lên hệ thống.
2. Nhấp nút "Import" hoặc nút xác nhận tải lên.

## Expected results
1. Hệ thống thực hiện import thành công toàn bộ sản phẩm trong file CSV.
2. Cơ sở dữ liệu lưu đúng các thông tin và thuộc tính của sản phẩm đã import (bao gồm bọc nháy kép chuẩn cho sản phẩm chứa dấu phẩy).
3. Giao diện hiển thị thông báo thành công: "Import hoàn tất: 2/2 sản phẩm được thêm".

## Actual results
1. Thất bại: Giao diện không xử lý dấu phẩy bọc trong dấu nháy kép đúng chuẩn RFC 4180. Dòng dữ liệu bị lệch cột khi gửi lên API (tên bị tách thành '"Bàn phím cơ', giá tiền nhận giá trị 'Keychron Q1"').
2. Thất bại: Cơ sở dữ liệu lưu sai thông tin do bị lệch cột.
3. Thành công: Giao diện hiển thị thông báo hoàn tất "Import hoàn tất: 2/2 sản phẩm được thêm" nhưng dữ liệu trong DB bị sai.

## Status
Failed

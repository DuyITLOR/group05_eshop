## Detail
| ID | Test Objective | File CSV (Input) | Database State (Output) | UI Report (Output) |
|---|---|---|---|---|
| FR16_EP_IV16 | Giao dịch nguyên tử (All-or-Nothing Rollback) khi có ít nhất một dòng dữ liệu bị lỗi trong số nhiều dòng dữ liệu đúng | File `.csv` (EXT1) với header đúng (HD1), chứa 4 dòng dữ liệu: <br>- Hàng 2: hợp lệ <br>- Hàng 3: hợp lệ <br>- Hàng 4: không hợp lệ (giá âm) (STATE2) <br>- Hàng 5: hợp lệ | Không có sản phẩm nào trong file (kể cả hàng 2, 3, 5) được lưu vào DB (DB2) | Hiển thị thông báo import thất bại: Chỉ rõ lý do lỗi tại Hàng 4, số dòng thành công hiển thị là 0 (REP3) |

## Preconditions
1. Người dùng đăng nhập hệ thống với tài khoản Admin (`role = 'admin'`).
2. Người dùng đang ở màn hình Import sản phẩm từ file CSV.

## Test data
| Field | Value |
|---|---|
| File Name | products_partial_error.csv |
| File Content | name,price,description,imageUrl,category_id<br>Laptop ASUS,15000000,Laptop văn phòng,,2<br>Điện thoại B,12000000,Điện thoại,,1<br>Sản phẩm lỗi,-50000,Mô tả,,1<br>Bàn phím cơ,3000000,Mô tả,,3 |

## Test steps
1. Chọn file `products_partial_error.csv` để tải lên hệ thống.
2. Nhấp nút "Import".

## Expected results
1. Hệ thống từ chối import toàn bộ file do phát hiện có lỗi ở Hàng 4 (giá âm) (Transaction Rollback).
2. Không có bất kỳ sản phẩm nào trong file (bao gồm các sản phẩm ở dòng 2, 3, 5) được thêm vào cơ sở dữ liệu.
3. Hiển thị thông báo import thất bại: Chỉ rõ lỗi tại Hàng 4, số dòng import thành công là 0.

## Actual results
1. Thất bại: Hệ thống không thực hiện rollback (giao dịch nguyên tử). Hàng số 4 bị lỗi giá âm vẫn được import thành công vào DB, các hàng 2, 3, 5 khác cũng được thêm bình thường vào cơ sở dữ liệu.
2. Thất bại: Các sản phẩm được lưu thành công vào DB thay vì rollback toàn bộ.
3. Thất bại: UI báo cáo import thành công 4/4 sản phẩm thay vì báo lỗi hàng 4 và rollback.

## Status
Failed

## Preconditions
1. Người dùng đã đăng nhập hệ thống với tài khoản User.
2. Tồn tại đơn hàng ở trạng thái `shipping` do chính User này đặt.

## Test data
| Field | Value |
|---|---|
| Token JWT | User Token hợp lệ (TOK_U1) |
| Order ID | Đơn hàng của chính mình (ORD_U1) |
| Trạng thái hiện tại | shipping (CS_U3) |

## Test steps
1. Kiểm tra sự tồn tại của nút "Hủy đơn hàng" đối với đơn hàng đang giao trên giao diện User.
2. Thử gửi yêu cầu hủy đơn hàng `shipping` qua API hủy.

## Expected results
1. Giao diện người dùng không hiển thị nút "Hủy đơn hàng" khi đơn hàng đã chuyển sang trạng thái `shipping`.
2. Hệ thống từ chối yêu cầu hủy qua API và trả về mã lỗi HTTP 400 Bad Request.
3. Trạng thái đơn hàng trong DB giữ nguyên là `shipping`.

## Actual results
1. Thất bại: Hệ thống vẫn cho phép người dùng tự hủy đơn hàng khi đơn hàng đang ở trạng thái `shipping`.
2. Thất bại: Trạng thái đơn hàng bị cập nhật thành `canceled` trong cơ sở dữ liệu và API trả về phản hồi thành công HTTP 200 OK thay vì báo lỗi (HTTP 400 Bad Request).

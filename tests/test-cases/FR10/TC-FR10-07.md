## Preconditions
1. Người dùng đăng nhập hệ thống với tài khoản User thường.
2. Tồn tại đơn hàng ở trạng thái `confirmed` do chính User này đặt.
3. Người dùng đang ở trang Lịch sử đơn hàng hoặc Chi tiết đơn hàng.

## Test data
| Field | Value |
|---|---|
| Token JWT | User Token hợp lệ (TOK_U1) |
| Order ID | Đơn hàng của chính mình (ORD_U1) |
| Trạng thái hiện tại | confirmed (CS_U2) |

## Test steps
1. Chọn đơn hàng có trạng thái `confirmed` trong danh sách đơn hàng cá nhân.
2. Nhấp chọn nút "Hủy đơn hàng".
3. Xác nhận hủy trên hộp thoại xác nhận.

## Expected results
1. Hệ thống thực hiện hủy đơn hàng thành công, hiển thị thông báo thành công phù hợp.
2. Trạng thái đơn hàng đổi thành `canceled` trên giao diện của người dùng.
3. Trạng thái đơn hàng được cập nhật thành `canceled` trong cơ sở dữ liệu.
4. API trả về phản hồi thành công (HTTP 200 OK).

## Actual results
1. Thành công: Hệ thống thực hiện hủy đơn hàng thành công, hiển thị thông báo thành công.
2. Trạng thái đơn hàng đổi thành `canceled` trên giao diện của người dùng và trong cơ sở dữ liệu.
3. API trả về phản hồi thành công (HTTP 200 OK).

# TC-FR10-01

## Requirement ID
FR10

## Module / Test type / Technique
Order State Machine / Functional / Equivalence Partitioning

## Test design source
test-design/EP-FR10.md

## Coverage
| Coverage Type | Covered IDs |
|---|---|
| Equivalence Partitioning | FR10_EP_V01 |

## Detail
| ID | Test Objective | State Transition (Input) | Kết quả (Output) |
|---|---|---|---|
| FR10_EP_V01 | Admin cập nhật trạng thái đơn hàng `pending` -> `confirmed` | `pending` -> `confirmed` (ST1) | Cập nhật thành công, trạng thái mới là `confirmed` (O1) |

## Preconditions
1. Người dùng đăng nhập hệ thống với tài khoản Admin (`role = 'admin'`).
2. Tồn tại một đơn hàng ở trạng thái `pending` trong hệ thống.
3. Người dùng đang ở màn hình Quản lý đơn hàng (Admin).

## Test data
| Field | Value |
|---|---|
| Trạng thái hiện tại | pending |
| Trạng thái cập nhật | confirmed |

## Test steps
1. Tìm đơn hàng có trạng thái `pending` trong danh sách đơn hàng.
2. Chọn đơn hàng đó và cập nhật trạng thái sang `confirmed`.
3. Xác nhận lưu thay đổi.

## Expected results
1. Trạng thái đơn hàng được cập nhật thành công thành `confirmed` trên giao diện Admin.
2. Trạng thái đơn hàng được cập nhật thành công thành `confirmed` trong cơ sở dữ liệu.
3. Trả về phản hồi thành công (HTTP 200 OK).

## Actual results
1. Thành công: Trạng thái đơn hàng được cập nhật thành `confirmed` trên giao diện Admin và trong cơ sở dữ liệu.
2. Thành công: API trả về phản hồi thành công (HTTP 200 OK).

## Status
Passed

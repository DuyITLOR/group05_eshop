## Detail
| ID | Test Objective | Token (Input) | Order ID (Input) | State Transition (Input) | Expected Output |
|---|---|---|---|---|---|
| FR10_EP_IV05 | Admin cập nhật từ trạng thái kết thúc `canceled` sang trạng thái khác (`canceled` -> `confirmed`) | Admin Token (TOK_A1) | Đơn hàng tồn tại (ORD1) | canceled -> confirmed (ST9) | - Báo lỗi chuyển đổi không hợp lệ (O2)<br>- Trả về HTTP 400 Bad Request |

## Preconditions
1. Người dùng đăng nhập hệ thống với tài khoản Admin (`role = 'admin'`).
2. Tồn tại một đơn hàng ở trạng thái kết thúc `canceled` trong hệ thống.
3. Người dùng đang ở màn hình Quản lý đơn hàng (Admin).

## Test data
| Field | Value |
|---|---|
| Token JWT | Admin Token hợp lệ (TOK_A1) |
| Order ID | Đơn hàng tồn tại (ORD1) |
| Cặp chuyển đổi | canceled -> confirmed (ST9) |

## Test steps
1. Tìm đơn hàng có trạng thái `canceled`.
2. Thử cập nhật trạng thái đơn hàng sang `confirmed` bằng cách gọi API với `status = 'confirmed'`.

## Expected results
1. Hệ thống từ chối cập nhật do `canceled` là trạng thái kết thúc.
2. Trả về mã lỗi HTTP 400 Bad Request.
3. Trạng thái đơn hàng trong DB giữ nguyên là `canceled`.

## Actual results
1. Thành công: Hệ thống từ chối cập nhật do `canceled` là trạng thái kết thúc.
2. Thành công: Trả về mã lỗi HTTP 400 Bad Request.
3. Thành công: Trạng thái đơn hàng giữ nguyên là `canceled`.

Passed

## Preconditions
1. Người dùng đăng nhập hệ thống với tài khoản Admin (`role = 'admin'`).
2. Tồn tại một đơn hàng ở trạng thái kết thúc `canceled` trong hệ thống.
3. Người dùng đang ở màn hình Quản lý đơn hàng (Admin).

## Test data
| Field | Value |
|---|---|
| Trạng thái hiện tại | canceled |
| Trạng thái cập nhật | confirmed |

## Test steps
1. Tìm đơn hàng có trạng thái `canceled` trong danh sách đơn hàng.
2. Thử cập nhật trạng thái đơn hàng sang `confirmed` bằng cách gọi API với `status = 'confirmed'`.
3. Xác nhận thực hiện.

## Expected results
1. Hệ thống từ chối cập nhật do `canceled` là trạng thái kết thúc (Final State).
2. Trả về thông báo lỗi phù hợp (HTTP 400 Bad Request).
3. Trạng thái đơn hàng trong DB giữ nguyên là `canceled`.

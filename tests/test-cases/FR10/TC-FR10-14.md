## Detail
| ID | Test Objective | Token (Input) | Order ID (Input) | State Transition (Input) | Expected Output |
|---|---|---|---|---|---|
| FR10_EP_IV07 | Người dùng thường thực hiện thao tác cập nhật của Admin | User Token (TOK_A2) | Đơn hàng tồn tại (ORD1) | pending -> confirmed (ST1) | - Báo lỗi không có quyền truy cập Admin (O5)<br>- Trả về HTTP 403 Forbidden |

## Preconditions
1. Người dùng đăng nhập hệ thống với tài khoản User thường.
2. Tồn tại một đơn hàng ở trạng thái `pending` trong hệ thống.

## Test data
| Field | Value |
|---|---|
| Token JWT | User Token thường (TOK_A2) |
| Order ID | Đơn hàng tồn tại (ORD1) |
| Cặp chuyển đổi | pending -> confirmed (ST1) |

## Test steps
1. Sử dụng User Token thường để gửi yêu cầu cập nhật trạng thái đơn hàng sang `confirmed` thông qua API Admin.

## Expected results
1. Hệ thống từ chối thực hiện do không có quyền truy cập Admin.
2. Trả về mã lỗi HTTP 403 Forbidden.
3. Trạng thái đơn hàng giữ nguyên là `pending`.

## Actual results
1. Thất bại: Hệ thống không kiểm tra vai trò admin của Token, cho phép tài khoản người dùng thường (`role = 'user'`) cập nhật thành công trạng thái đơn hàng qua API Admin.
2. Thất bại: Trạng thái đơn hàng bị thay đổi thành `confirmed` và API trả về phản hồi thành công (HTTP 200 OK) thay vì bị chặn (HTTP 403 Forbidden).

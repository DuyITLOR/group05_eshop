# TC-D9-07

## Requirement ID
FR-20

## Module / Test type / Technique
Order Cancel (Mobile) / Functional / Equivalence Partitioning

## Test design source
test-design/EP-D9.md

## Coverage
| Coverage Type | Covered IDs |
|---|---|
| Equivalence Partitioning | FR20_EP_IV03 |

## Preconditions
1. Người dùng đã đăng nhập tài khoản khách hàng trên ứng dụng Mobile.
2. Người dùng có đơn hàng ở trạng thái `canceled` (Đã hủy).

## Test data
| Field | Value |
|---|---|
| Trạng thái đơn hàng | canceled |

## Test steps
1. Trên ứng dụng Mobile, người dùng chuyển đến tab/giao diện "Hồ sơ".
2. Cuộn xuống phần "Lịch sử đơn hàng".
3. Tìm đến đơn hàng có trạng thái "Đã hủy" (`canceled`).

## Expected results
1. Nút "Hủy đơn" không được hiển thị tại đơn hàng này.

## Actual results
1. Thành công: Ẩn nút "Hủy đơn" trên ứng dụng Mobile khi đơn hàng ở trạng thái "Đã hủy" (`canceled`).

## Status
Passed

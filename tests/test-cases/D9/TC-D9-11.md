## Preconditions
1. Người dùng đã đăng nhập tài khoản khách hàng của mình.
2. Người dùng có đơn hàng ở trạng thái `shipping` (Đang giao).

## Test data
| Field | Value |
|---|---|
| Order ID | ID của đơn hàng ở trạng thái shipping của người dùng |
| Token xác thực | JWT Token hợp lệ của người dùng |

## Test steps
1. Gửi yêu cầu HTTP PUT tới API `/api/orders/{orderId}/cancel` để cố gắng hủy đơn hàng đang ở trạng thái `shipping`.

## Expected results
1. Hệ thống từ chối yêu cầu hủy và trả về lỗi HTTP 400 Bad Request kèm thông báo lỗi phù hợp (VD: "Cannot cancel this order" hoặc "Không thể hủy đơn").
2. Đơn hàng trong CSDL vẫn giữ nguyên trạng thái cũ là `shipping`.

## Actual results
1. Thất bại: Hệ thống phản hồi thành công với HTTP 200 OK và trả về {"message":"Order canceled successfully"} thay vì chặn yêu cầu với HTTP 400 Bad Request.
2. Thất bại: Trạng thái đơn hàng trong CSDL bị thay đổi từ `shipping` thành `canceled` (Lỗi logic nghiệp vụ).

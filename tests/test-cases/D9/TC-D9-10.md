# TC-D9-10

## Requirement ID
FR-20

## Module / Test type / Technique
Order Cancel (Mobile) / Functional / Equivalence Partitioning

## Test design source
test-design/EP-D9.md

## Coverage
| Coverage Type | Covered IDs |
|---|---|
| Equivalence Partitioning | FR20_EP_IV06 |

## Preconditions
1. Người dùng A đã đăng nhập tài khoản khách hàng của mình.
2. Có đơn hàng thuộc sở hữu của người dùng B hoặc một ID đơn hàng không tồn tại trên hệ thống.

## Test data
| Field | Value |
|---|---|
| Order ID | ID của đơn hàng thuộc về người dùng B hoặc ID không tồn tại |
| Token xác thực | JWT Token hợp lệ của người dùng A |

## Test steps
1. Gửi yêu cầu HTTP PUT tới API `/api/orders/{orderId}/cancel` kèm theo JWT token của người dùng A, nhưng chỉ định `orderId` là của người dùng B hoặc một ID không tồn tại.

## Expected results
1. Hệ thống từ chối yêu cầu và trả về lỗi HTTP 404 Not Found kèm thông báo lỗi phù hợp (VD: "Order not found").
2. Trạng thái đơn hàng trong CSDL không thay đổi.

## Actual results
1. Thành công: Hệ thống phản hồi HTTP 404 Not Found và trả về `{"error":"Order not found"}`.
2. Thành công: Trạng thái đơn hàng trong CSDL được giữ nguyên.

## Status
Passed

| FR20_EP_IV08 | Người dùng cố gắng hủy đơn hàng đã `delivered` bằng API | Hợp lệ (AUTH1) | Của người dùng (OWN1) | `delivered` (ST4) | - Trả về HTTP 400 Bad Request và thông báo lỗi không được phép hủy (OUT3)<br>- Giữ nguyên trạng thái đơn hàng cũ là `delivered` |

## Preconditions
1. Người dùng đã đăng nhập tài khoản khách hàng của mình.
2. Người dùng có đơn hàng ở trạng thái `delivered` (Đã giao).

## Test data
| Field | Value |
|---|---|
| Order ID | ID của đơn hàng ở trạng thái delivered của người dùng |
| Token xác thực | JWT Token hợp lệ của người dùng |

## Test steps
1. Gửi yêu cầu HTTP PUT tới API `/api/orders/{orderId}/cancel` để cố gắng hủy đơn hàng đã ở trạng thái kết thúc `delivered`.

## Expected results
1. Hệ thống từ chối yêu cầu hủy và trả về lỗi HTTP 400 Bad Request kèm thông báo lỗi phù hợp (VD: "Cannot cancel this order" hoặc "Không thể hủy đơn").
2. Đơn hàng trong CSDL vẫn giữ nguyên trạng thái cũ là `delivered`.

## Actual results
1. Thành công: Hệ thống phản hồi HTTP 400 Bad Request và trả về `{"error":"Cannot cancel this order."}`.
2. Thành công: Trạng thái đơn hàng trong CSDL được giữ nguyên.
